

1. Project Overview

Testing the functional flow of the MidBrains Technologies website, including navigation and lead generation.

3. Detailed Functional Requirements
- Homepage: Loading speed, Title, URL validation.
- Navigation: All menu links (MERN, Testing, etc.) should work.
- Form Submission: Contact form and Lead form functionality.
- UI Elements: Buttons and Call-to-Action (CTA) responsiveness.

3. Technical Requirements & Tools
- Tool: Selenium with Java.
- Browser: Google Chrome.
- Framework: Page Object Model (POM).
- Wait Mechanism: Explicit Waits

4. Requirement Traceability (RTM)
- Including navigation and lead generation.
- Requirement: Contact Form Validation.

5. Assumptions & Constraints
- Assumptions:
- System Availability: The website is stable and "Up and Running" during the testing phase.
- Resource Access: All necessary URLs and form details are provided and accessible.

- Constraints:
- Scope Limitation: Testing is limited to Desktop browsers and Mobile Responsiveness (UI layout) only.
- Functional Exclusions: Payment Gateway and Backend/Database testing are strictly Out of Scope.
- Tooling: Automation is restricted to the specified tools (Selenium/Playwright) as per the assignment.
